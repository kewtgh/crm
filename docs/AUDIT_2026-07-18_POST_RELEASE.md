# Lumina CRM v1.0.0 发布后审计

审计日期：2026-07-18
审计范围：应用架构、核心业务流程、数据口径、安全边界、UI/UX、可访问性、国际化、开发体验、自动化测试与 Sites 发布状态。

## 结论

项目的身份认证、权限校验、RLS、请求来源校验、Webhook 签名与幂等、CSP、构建和基础测试具备良好基础，但 v1.0.0 仍存在一个核心销售流程阻断问题，以及多项会造成数据遗漏、指标误导、环境不可复现或键盘操作不完整的问题。生产发布前必须完成本报告 P0/P1 项，并执行完整浏览器、数据库和运行时验收。

## 已验证的正向结果

- TypeScript、ESLint 和生产构建通过。
- 17 个 Node 测试通过。
- npm 依赖安全审计未发现已知漏洞。
- 数据库 Schema lint 通过。
- v0.9 HTTP、Phase 2 和 v0.9 业务烟测通过。
- 绝大多数变更接口执行来源校验，外部 Webhook 使用签名边界。
- 服务端角色、AAL2、RLS、审计日志、Webhook 重放与幂等保护已落地。

## 发现

### P0：商机阶段转换不可用

商机看板只提交 `stage` 和 `probability`，但更新接口要求活跃阶段必须有预计成交日期和中英文下一步行动，Won 必须有成交凭证，Lost 必须有失败原因。用户在界面中选择任何目标阶段都会被接口拒绝。

同时，新建商机表单未把预计成交日期和下一步行动标为必填，但接口要求这些字段必填。

### P1：财务总览静默截断业务数据

合同、应收和付款最多读取 50 条，退款和对账最多读取 30 条。页面基于这些截断数组计算待收款、退款和对账异常，可能漏掉真正需要处理的财务风险。对账请求失败还会回退为空数组，使系统故障看起来像“没有异常”。

### P1：学校、联系人、任务新增流程不完整

三个模块共用仅含中英文名称和联系方式的通用表单，无法录入学校城市/课程体系、联系人职位/所属机构、任务截止日期/优先级/负责人/关联对象等关键数据。

### P1：统计口径混用

学校、联系人和任务只加载前 5 条数据，却使用这 5 条计算“需要关注”和“平均完整度”，同时展示全局总数。商机漏斗统计在搜索、翻页、新增或阶段切换后不刷新。

### P1：用户时区和日期格式未全局生效

设置页可保存时区和日期格式，但多数客户端日期仍使用浏览器时区，仪表盘数据库函数使用数据库 `current_date`/`now()`。用户设置与报表口径可能在跨日或跨月时产生偏差。

### P1：本地环境配置和运行时烟测不可复现

本地配置脚本会覆盖 `.env.local`，但不生成当前版本所需的工作区、独立登录限流、Webhook、Worker 和集成配置。v1 HTTP Webhook 烟测因此无法执行。

### P1：质量门禁存在顺序依赖

`smoke:v09` 写入 Worker 心跳但未清理。按照 README 推荐顺序先执行烟测、再执行数据库测试时，readiness 测试会因残留心跳从期望的 6 个缺失 Worker 变成 5 个，导致 132 个数据库行为测试中 1 个失败。

### P1：请求客户端没有统一

组件中仍存在 60 余处直接 `fetch`，只有少数页面使用统一的 `apiFetch`。超时、Abort、会话刷新、错误码、request ID 和错误文案处理不一致。

### P1/P2：弹层和移动导航可访问性不完整

共享抽屉已经支持焦点陷阱、Escape 和焦点恢复，但商机、日历、通用新增、销售分析等页面仍使用独立实现。移动导航没有完整的 Escape、焦点陷阱、焦点恢复和背景滚动锁定；部分 `role="menu"` 子项缺少对应菜单语义。

### P2：设计 Token 漂移

全局 CSS 使用了 `--border`、`--mint-soft`、`--soft`、`--surface-soft`、`--teal`、`--text` 等未定义变量。浏览器会丢弃相关声明，造成边框、颜色和背景不稳定。

### P2：分页和空数据错误

数据表固定每页 5 条；非法 `page` 查询参数会产生 `NaN`；筛选后当前页超过总页数时可能展示空列表。部分服务器请求将失败转换为空数据，掩盖运行故障。

## 发布状态

- Sites 项目已经存在，但审计时没有已保存版本和生产部署。
- 外部服务、Worker 调度、生产心跳、Webhook 密钥、域名和生产环境变量仍需发布验收。
- 审计会话没有可用浏览器实例，因此桌面、平板、375px 移动端、键盘焦点和视觉回归尚未形成可执行证据。

## 建议引入的产品能力

1. 商机阶段引导器：按阶段收集必填资料、Won 凭证、Lost 原因和下一步行动。
2. 财务风险中心：集中呈现逾期应收、部分付款、退款积压和对账异常。
3. 数据完整度工作台：识别并快速补齐学校、联系人和任务的缺失字段。
4. 销售自动化：长期未跟进、预计成交临近、续约和逾期付款自动生成下一步行动。
5. 保存视图和团队仪表盘：保存筛选、排序、每页条数和业务视图。
6. 发布与运行控制台：统一展示 Worker 心跳、Webhook 堆积、环境配置和质量门禁状态。

## 审计限制

本报告基于源码、数据库迁移、静态契约、构建、自动化测试和本地运行时检查。浏览器实例不可用导致像素级和真实辅助技术验收未完成；该项必须在最终发布前补做。
