# Lumina Education CRM v1.0.0 完整修改计划

计划日期：2026-07-18  
输入：`AUDIT_2026-07-18_V091.md`  
目标版本：1.0.0  
目标：关闭补充审计中的全部仓库内可执行缺口，并将外部环境限制收敛为明确、可重复验证的发布门禁。

## 第一阶段：身份、租户和 API 安全

### 1.1 员工停用一致性

- 替换旧 Auth membership 触发器：仅在创建时初始化 membership，更新 metadata 时同步角色但不得覆盖显式停用。
- 停用员工时同时更新 membership、Auth ban 和可撤销会话版本。
- 身份同步失败写入可重试 repair queue，管理员可查看和重试。
- 移除代码中的固定 workspace UUID，统一由受控配置或数据库当前 workspace 决定。

验收：

- 停用后 membership 保持 `SUSPENDED`。
- 旧会话访问 API 和 PostgREST 均被拒绝。
- 失败补偿不会被标记为同步成功。

### 1.2 租户隔离和敏感写操作

- 所有运维重试按当前 workspace 限定并写审计。
- 所有新增写接口统一执行可信来源校验。
- 产品目录、去重合并、人工运维和身份操作统一执行 AAL2。
- 权限解释器可解释 `MFA_REQUIRED`、workspace、owner/team 与对象状态原因。

### 1.3 Webhook 可信信封

- 签名覆盖版本、provider、event ID、event type、timestamp 和请求体摘要。
- 强制时间窗口，记录 nonce/事件幂等键，拒绝过期、未来或篡改信封。
- 对不支持统一 HMAC 的供应商保留 provider adapter 接口。

## 第二阶段：Worker 和运维闭环

### 2.1 统一租约

- 通知、提醒、日历、生成任务和 Webhook 增加 `locked_at`、`lease_expires_at`、`locked_by`。
- claim 时回收已过期租约；complete/fail 仅允许当前 lease token 完成。
- 保留指数退避、最大尝试次数和明确死信状态。

### 2.2 人工重放与审计

- 运维重试支持所有死信队列，包括 Webhook。
- 重试只允许当前 workspace，写入 actor、对象、旧状态和新状态。
- repair queue、死信与卡死任务在统一运维中心可见。

### 2.3 Readiness 和 SLA

- readiness 纳入全部 Worker、提醒、Webhook、导入和审批积压。
- 队列配置 SLA 分钟数，返回积压、失败、卡死、超限和最老任务。
- 只有关键 Worker 新鲜、无关键死信、数据库/Auth 正常时才 ready。

## 第三阶段：数据合并和业务完整性

### 3.1 安全去重合并

- 预览只返回并允许选择实际可合并字段。
- Consent 按“更严格状态优先、最新证据保留、完整审计”合并。
- 迁移联系人/组织的全部已知关联，包括 NBA。
- SQL 对未知字段选择拒绝，不再静默忽略。
- 合并前后关系数量、冲突和最终值可验证。

### 3.2 续约 Playbook

- 编辑前加载已有 Playbook。
- 根据合同到期日自动建议 90/60/30/14/7 天阶段、风险和 SLA。
- 增加健康分、逾期、负责人和最近互动证据。
- 合同列表和 KPI 全部使用服务端完整摘要，移除固定数字。

### 3.3 产品包和汇率

- 产品包增加版本、生效区间和不可变历史。
- 报价可引用产品包和包版本，并持久化展开后的条目。
- 数据库在报价提交时执行包级折扣上限。
- 报价锁定汇率快照与本位币金额；历史报价、合同和付款不使用实时汇率回写。

### 3.4 集成中心

- 增加管理员可控的连接配置、启用/停用、重新授权状态和手工同步请求。
- 同步任务进入统一队列并推进游标、最近成功、错误和重试。
- 电子签名和会计 Webhook 采用显式 adapter 处理并保持审计；无凭据时继续显示真实未连接。

### 3.5 Next Best Action

- 增加生成批次、适用/不适用结果和原因。
- 支持定时生成、规则版本、证据、采纳、驳回及任务完成关联。
- 分析采纳率、拒绝原因、完成率和规则效果。

## 第四阶段：API、UI/UX 和分析

### 4.1 统一 API 客户端

- `apiFetch()` 默认超时，支持 AbortSignal 合并、结构化错误和一次会话刷新。
- 新功能页面全部迁移到统一客户端。
- 错误态保留失败模块，不把网络失败显示成空数据。

### 4.2 可访问抽屉

- 产品、续约、客户活动、导入和运维交互迁移到共享 Drawer。
- 375px 无横向溢出，移动端字段保留可辨识标签。

### 4.3 经营分析

- 增加续约漏斗与逾期 SLA。
- 增加客户留存/流失趋势。
- 增加预测值与实际成交的准确率。
- 增加队列 SLA 达标率。
- 增加 NBA 采纳、拒绝和任务完成转化。
