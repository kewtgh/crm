# Lumina Education CRM 综合审计报告

- 审计日期：2026-07-19
- 审计基线：`def332c`（v1.1.0）
- 审计范围：架构、权限、业务闭环、UI/UX、无障碍、双语、数据导入、运维、测试与托管
- 结论：工程基线可构建且自动化检查通过，但业务闭环、真实浏览器质量门禁和生产运行条件仍未全部满足。

## 已验证基线

- TypeScript 类型检查通过。
- ESLint 通过。
- Vinext 生产构建通过。
- Node 测试 21/21 通过。
- pgTAP 数据库测试 140 项通过。
- phase2、HTTP v0.9、HTTP v1.0、v0.9、v1.1 共五组 smoke 测试通过。
- 数据库 lint 无 schema 错误。
- `/api/health` liveness 返回 200。
- 审计前后 Git 工作区干净。

## 发布阻断项

### REL-01：readiness 失败

readiness 返回 503，六类 worker 缺失或心跳过期，并存在历史待处理任务。生产发布必须启动调度器、保证 worker 心跳和积压告警，使 readiness 稳定返回 200。

### REL-02：生产托管未落地

托管项目存在与基线代码对应的保存版本，但没有生产部署、域名、生产环境变量或在线 URL。

### REL-03：缺少真实浏览器回归

现有 `tests/rendered-html.test.mjs` 是源码断言，并不渲染 DOM。没有覆盖桌面端、平板、手机、键盘、焦点、溢出、竞态和 axe 规则的真实 E2E。此次审计的浏览器连接器因本机 Node 运行时冲突未能运行，因此所有 UI/无障碍结论均为代码级审计，不替代实机验收。

## 高优先级缺陷

### CRM-01：任务负责人选择与 RLS 冲突

创建任务界面允许选择其他负责人，服务端也接受 `ownerId`，但数据库 insert policy 仅允许 `owner_id = auth.uid()`。选择同事后提交必然失败。

### CRM-02：核心 CRM 缺少更新闭环

学校、联系人和普通任务只有列表、详情和创建路径，缺少通用字段更新、归档和变更历史入口。数据质量页面可以跳到记录，却不能直接修复源字段，也允许在未修复数据时手工关闭问题。

### CRM-03：列表导出入口永久禁用

学校、联系人、任务页面展示了导出按钮，但按钮永久 disabled，无法发起审批、查看状态或下载批准后的产物。

### ERR-01：后端错误被伪装为空数据

导入、重复数据和数据质量的服务端加载 catch 会返回空数组，数据库或权限故障会被显示为“暂无数据”。全站也缺少统一的 `error.tsx`、`global-error.tsx`、`loading.tsx` 和 `not-found.tsx`。

### SEC-01：密码重置缺少应用层防滥用

密码重置只做 Origin 和邮箱格式校验，未复用登录接口的持久限流与 Turnstile；上游 429 被转换成普通 502，无法保留 `Retry-After`。

## UI、无障碍与一致性缺陷

### UI-01：移动端顶栏 CSS 级联冲突

移动端媒体查询把顶栏设为 60px，但后置全局规则覆盖成更高尺寸，而弹层偏移仍使用旧高度，造成遮挡与错位风险。

### A11Y-01：可搜索选择器 ARIA 模型不完整

实际焦点进入搜索输入框后，`aria-activedescendant` 仍位于失焦按钮；键盘活动项也不会滚入视口。

### A11Y-02：顶栏弹层缺少完整焦点模型

通知 dialog 和用户 menu 缺少打开聚焦、焦点约束、方向键/Home/End 菜单导航及关闭后的焦点恢复。全站没有 skip-to-content。

### UI-02：服务端与客户端默认排序不一致

服务端无 direction 时默认倒序，客户端默认正序，hydration 后可能立即重排甚至切换首屏记录。

### UI-03：状态筛选与保存视图缺少 schema

状态使用循环按钮，用户无法一次看到全部选项；无记录时可能显示原始状态码。localStorage 保存视图只校验 name，无法安全处理旧版本或篡改值。

### I18N-01：中文界面泄露技术码

合同、活动、导入、合并、运维、产品和设置页面仍显示 `NOT_STARTED`、`LOW`、`CALL`、`TARGET/SOURCE`、字段键和资源代码。

### UX-01：重复记录合并要求手工输入 UUID

用户必须粘贴两个 36 位 ID；缺少可搜索记录选择、名称与关键字段上下文。合并和其他危险操作也缺少统一 pending 锁、确认和幂等键。

### IMP-01：CSV 解析不兼容完整 CSV 语法

解析器按物理行拆分，不能处理引号中的换行，也不校验未闭合引号、编码、分隔符和 500 行 API 上限。

### SEARCH-01：“全局搜索”覆盖范围不足

当前仅搜索组织、联系人和用户，不包含机会、任务、合同、报价和产品，能力与名称不一致。

## 架构与可维护性

- 多个页面重复实现分页、筛选、URL 同步、加载、错误和重试，行为已经分叉。
- 多个远程搜索没有统一的取消和 last-request-wins 保护。
- CSS 依赖追加式覆盖和 `!important` 最终保护，存在持续级联回归风险。
- 大部分 CRM 路由没有独立、双语页面 metadata。
- 部署文档停留在迁移 `037`，仓库已经包含 `038`；旧审计中的数据库测试数量也已经过期。
- Vinext 构建成功，但路由分类仍输出多项 `Unknown` 提示，需要在部署目标上持续验证。

## 建议纳入的产品能力

1. 学校、联系人、任务的编辑、归档、版本历史与数据质量修复闭环。
2. 团队任务委派、工作量/容量视图、SLA、超时提醒和主管工作队列。
3. 权限感知的全局命令搜索，覆盖客户、机会、任务、合同、报价、产品及常用动作。
4. 团队共享视图、批量操作预览、导出审批和幂等执行。
5. 邮件、日历和通知提供商的真实接入、失败重试和运维告警。
6. 在数据质量稳定后提供人工确认式 AI 辅助：沟通摘要、下一步行动提取、重复数据建议和销售风险提示。

## 审计边界

本报告不把源码审查等同于真实浏览器验收；发布前必须完成实施计划中的浏览器矩阵。外部邮件、日历、通知和 AI 提供商还需要真实凭据与业务方授权，代码可以完成安全适配层和可观测性，但生产连通性必须由部署环境验收。
