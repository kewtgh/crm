# Lumina Education CRM v0.8.0 完整开发计划

计划日期：2026-07-17
输入：第二阶段审计、0.7.0 整改路线图 P1、五项新增业务要求

## 完成定义

每项能力必须同时具备真实数据库持久化、RLS、审计、服务端校验、搜索/分页、中英文、移动端可用性、就地错误、空/载入/失败状态和自动化测试。只显示界面、只写计划或返回假成功均不算完成。

## A. 数据与权限基线

1. 新增联系人同意、报价及版本、应收计划、退款、对账、导入批次/行/重复候选、日历送达等规范化表。
2. 为新表建立 workspace、负责人/关联组织范围 RLS；管理员敏感操作继续要求 AAL2。
3. 为业务写操作提供受控 RPC，增加审计触发器、唯一键、状态约束和分页索引。
4. 客户时间线只联合现有业务表和新增规范化流水，不建立汇总影子客户表。

## B. 客户 360 时间线

1. 建立统一事件投影：组织、联系人、商机、任务、活动、预约、合同、付款、关系证据和审批。
2. 支持类型筛选、日期排序、服务端分页、来源对象链接和中英文标题。
3. 在客户组织列表提供明确的 360 入口；详情页同时展示概览、风险和时间线。

验收：无权限用户不能通过时间线看到原本不可见记录；翻页不重复、不漏项；不产生双写表。

## C. 联系人同意与沟通偏好

1. 支持 EMAIL、SMS、PHONE、WECHAT、WHATSAPP 等渠道与 MARKETING、SERVICE、TRANSACTIONAL、EVENT 用途组合。
2. 记录授权状态、来源、证明说明、取得时间、撤回时间、保留期限、勿扰时段和全局勿扰。
3. 联系人详情内管理偏好；撤回后保留历史证据，不物理删除。
4. 营销触达、营销导出和外部日历邀请在服务端调用统一资格检查。

验收：无有效授权、已撤回、已过保留期或处于全局勿扰的营销操作被拒绝，并在操作位置显示原因。

## D. 报价—合同—收款—退款—对账

1. 报价支持不可变版本、有效期、明细金额、折扣、条款和状态迁移。
2. 超出折扣边界时创建管理员审批；审批通过前不能接受或转换合同。
3. 合同建立应收计划；多次付款更新已收金额和部分付款状态。
4. 退款使用独立记录、审批、金额上限和凭证引用，不覆盖原付款金额。
5. 对账记录预期、实际、差异、原因、处理人和解决状态。

验收：任一时点可从报价版本、合同快照、应收、付款、退款与对账记录重算金额；禁止超额付款和超额退款。

## E. 数据质量、导入与查重闭环

1. 导入创建批次，保存字段映射、文件摘要和幂等键；支持 CSV 预检。
2. 逐行标准化、必填/格式校验和重复检测；候选按邮箱、电话和双语名称给出证据与置信度。
3. 人工选择 CREATE、UPDATE、MERGE 或 SKIP；高风险合并不自动执行。
4. 小批次幂等执行，保存每行结果；批次回滚只撤销本批次创建/更新并校验后续修改冲突。
5. 数据质量中心以真实规则运行结果展示问题、负责人和解决状态。

验收：重复提交不重复建档；单行失败不阻塞其他有效行；错误显示在对应行；回滚留审计且不破坏批次外修改。

## F. 日历外部送达

1. 预约保存外部参与人；创建、变更、取消生成独立送达任务和事件版本。
2. 邮件 worker 调用配置的受控 webhook，保存 QUEUED、SENDING、DELIVERED、FAILED、CANCELLED 状态。
3. 使用幂等键和指数退避；达到上限后显示最终错误并允许管理员重试。
4. 日历详情展示逐收件人送达状态，不在服务确认前使用“已发送”。

验收：创建/变更/取消模板和状态可区分；失败可重试且不会重复发送同一版本。

## G. UI/UX 与国际化收尾

1. 启用已完成的数据质量、导入和查重入口；删除不属于当前 CRM 范围的学生、家庭、升学、独立线索和 AI 占位路由。
2. 管理门户使用真实当前用户和动态 MFA 数量，删除固定示例。
3. 删除未使用演示文案，新增可见文本、错误、状态和无障碍标签全部覆盖中英文。
4. 复用搜索、分页、状态、抽屉、表单和就地消息组件；手机端不横向溢出。

## H. 测试、部署和回滚

1. 数据库测试覆盖 RLS、同意门禁、金额不变量、状态迁移、导入幂等/回滚和送达重试。
2. 类型、Lint、生产构建和页面契约测试全部通过。
3. Chromium 1228 验收覆盖桌面、375px、键盘、弹窗焦点、分页和错误位置；若运行时不可用则明确阻止生产发布。
4. 部署顺序：备份、应用迁移、配置邮件 webhook、启动 reminder/outbox/calendar/export worker、冒烟、私有发布；失败按新增迁移的向前修复方案处理。

## 完成记录

完成日期：2026-07-17

- A–G 已实现：迁移 `021`–`024`、RLS/审计/RPC、API、响应式页面和完整 zh-CN/en 目录已经落盘。`023` 修复报价版本字段歧义；`024` 修复导入无重复候选时的空数组边界。
- 客户 360 从真实来源表实时投影，并复用现有组织权限；没有创建影子客户表。
- 联系人同意由统一数据库门禁执行；营销联系人导出需要审批，导出 worker 在生成文件时按最新同意状态再次筛选。
- 财务中心覆盖报价版本、折扣审批、合同转换、应收计划、部分付款、退款审批/凭证和对账差异。
- 导入中心覆盖 CSV 映射、预检、重复候选、人工决策、小批次处理、逐行错误、幂等提交和冲突保护回滚；数据质量中心运行真实规则。
- 日历邀请、变更和取消生成逐收件人的送达任务；worker 只有收到提供商成功响应才标记送达，失败使用幂等键和指数退避。
- 验证通过：TypeScript、ESLint、生产构建、15 项 Node 契约测试、27 项 pgTAP，以及真实本地 Supabase 的 phase-two 业务冒烟。
