# 架构、UI/UX 与指令中断审计

版本：0.4.0
审计日期：2026-07-16

## 审计结论

本轮在 v0.2.0 基线上完成了用户追加的日历、销售、合同、产品、消费分析、关系推进、关单建议、真实国际化和独立身份模型，并重新检查了多次追加指令是否让较早任务中断。已发现并补齐的中断缺口包括：旧后台/运营/设置页尚未接入语言目录、销售和首页残留硬编码界面文案、产品自定义计费保存了翻译文本、旧架构报告仍把国际化列为未完成。

本次又将实现与原始压缩包中的 `01_PRD.md`、`02_PAGE_INFORMATION_ARCHITECTURE.md` 和数据模型逐项比对。基于新增业务要求，权限模型明确为超级管理员、普通管理员，以及销售总监、销售经理、销售专员和独立的销售支持支线；没有重新引入来源不明的其他项目角色。学生、家庭、监护关系和升年级在原始 PRD 中有明确领域定义，因此继续保留。

当前版本适合作为受保护的 UI、认证和交互验收环境。它不是可承载真实学生、家庭和客户资料的生产 CRM；业务记录与大多数写操作仍是 fixture 或会话内原型。界面不会把这些原型误写成生产完成状态。

## 仍是上线阻断项（P0）

1. **业务数据层与工作区隔离**：建立 workspace、membership、核心业务表、审计表、附件存储及 RLS；当前 fixture 不得导入真实敏感资料。
2. **服务端分页与查重**：真实列表统一返回 `{ items, total, page/cursor, pageSize }`，设置最大页大小；查重在服务端事务内重验。
3. **完整权限矩阵**：管理员和销售层级已进入认证白名单，审批/分配已有 RLS；学校、家庭、学生及其他业务对象仍需 workspace 范围策略。
4. **真实设置和安全服务**：头像、通知、隐私导出、密码、其他设备退出、MFA 状态及恢复代码仍是明确标注的验收界面。
5. **后台任务**：日程通知、合同续约、目标版本、导入和批处理需要幂等 worker、重试、死信和审计。
6. **Chromium 1228 回归**：本次环境没有可用的应用内 Chromium 实例，无法完成真实点击、截图和 375px 视觉验证；部署前必须按 `DEPLOYMENT.md` 清单执行。

## 克制的下一阶段架构

```text
Browser
  -> Vinext/Next BFF（会话、校验、权限、限流）
      -> Supabase Auth（身份、MFA）
      -> PostgreSQL + RLS（workspace、业务数据、审计、服务端分页）
      -> Storage（头像、附件）
      -> Worker（提醒、导入、续约、批处理）
```

不建议为了“现代化”改成复杂微服务或 monorepo。先选择“学校 + 联系人 + 任务”作为真实垂直切片，完成 CRUD、查重、RLS、审计和分页，再按域替换 fixture；这与当前风险和修改幅度相匹配。

## UI/UX 复检

- 导航层级、卡片、颜色、图标、搜索、分页、消息和表单反馈已统一；状态不只依赖颜色。
- 手机断点已覆盖日历、销售 Playbook、合同、产品和分析页；宽表仍允许横向滚动，生产阶段应按真实列宽决定是否改为摘要卡片，避免一次性重写所有表格。
- 登录、注册、密码和设置错误在对应表单附近；错误原因不进入 URL。
- 人员姓名保持“中文 / English”同时显示；学校、产品、菜单和说明只随当前语言显示。
- 抽屉支持 Escape 和焦点返回的页面已覆盖新增核心流程；其他旧菜单的完整焦点圈定仍应纳入浏览器回归。
