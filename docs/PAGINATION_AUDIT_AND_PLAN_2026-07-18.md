# 全站列表分页审计与实施计划

审计日期：2026-07-18  
范围：CRM 业务页面、管理页面、运营页面、列表 API、仓储查询、共享 UI 和响应式样式。

## 审计结论

审计发现分页行为此前存在三类不一致：

1. 多个页面分别使用 5、8、15、20、24、25 或 50 条作为默认值，且部分页面自行绘制数量选择器。
2. 产品包、汇率快照、续约提醒、近期日程、销售成员和关系矩阵等增长型列表只截取前若干项或一次性渲染全部数据。
3. 运维重试队列与 Next Best Action 最多读取 100 条，超过上限的数据会被静默隐藏，也没有可操作的翻页入口。
4. 最终门禁发现本日新增的构建工具链安全公告；Cloudflare Vite 插件、Vite、Wrangler、PostCSS 及其传递依赖需要安全升级。

全站现统一复用 `Pagination`，每页数量只提供 **10 / 20 / 50**。切换每页数量时回到第一页；服务端列表重新查询，当前页面内的快照列表重新切片。

## 列表分类与处理结果

| 分类 | 列表 | 处理 |
| --- | --- | --- |
| 核心 CRM | 学校、联系人、任务及通用资源表 | 保留服务端搜索与分页，改用共享 10 / 20 / 50 选择器 |
| 销售与合同 | 商机、合同、续约提醒、销售成员、重点客户关系矩阵 | 商机和合同服务端分页；提醒和分析快照使用共享客户端分页 |
| 财务 | 报价、合同计划、应收、付款、退款、对账 | 各区块独立分页并共享同一每页数量 |
| 治理与后台 | 审批、绩效分配、安全审计、员工、导出 | 统一默认值与共享选择器；服务端列表保留精确总数 |
| 数据运营 | 导入批次、导入行、数据质量、通知、客户 360 时间线 | 统一共享分页；导入行支持 10 / 20 / 50 |
| 产品与日程 | 产品、产品包、汇率快照、近期日程 | 移除固定截取显示，增加共享分页 |
| 运维 | 可重试任务、Next Best Action | 新增真实服务端分页、精确总数和数据库分页 RPC；移除 100 条静默上限 |

以下内容不属于增长型业务列表，因此不添加分页：

- 仪表盘、通知弹层和管理首页中的摘要预览；它们已有“查看全部”入口。
- 日历单元格中的当日事件摘要；完整近期事件列表已分页。
- 队列类型、Worker 类型、集成提供商、角色、状态、渠道和通知偏好等固定枚举。
- 联系人同意状态；它由有限的“渠道 × 用途”当前状态组合构成。
- 趋势、漏斗、Top N 客户和产品组合等有明确分析窗口或排名上限的图表数据。

## 验收标准

1. 任一共享分页器均显示总数、页码、上一页/下一页和 10 / 20 / 50 数量选项。
2. 数量变化后回到第一页，不出现旧页越界、空白页或仍按旧数量请求的问题。
3. 服务端增长型列表返回精确 `total`、`page` 和 `pageSize`。
4. 运维列表超过 100 条时仍可继续翻页，不再静默丢失。
5. 小屏幕下数量选择器与页码可换行，且原生 `select` 保留键盘与辅助技术可用性。
