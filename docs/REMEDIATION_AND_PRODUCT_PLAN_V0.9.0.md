# Lumina Education CRM v0.9.0 修复与产品完善计划

计划日期：2026-07-17  
目标版本：0.9.0  
输入：`AUDIT_2026-07-17_FINAL.md`

## 总体目标

将 v0.8.0 从“可构建的业务原型”提升到“数据库权限闭环、业务状态一致、失败可恢复、运营可观察、界面不误导、测试可证明”的候选发布版本。在没有外部供应商凭据时完成真实的集成基础设施和未连接状态，不伪造第三方连接或发送结果。

## 第一阶段：数据库最终授权

### 任务

- 撤销并删除旧 `refund_payment`。
- 撤销 `approval_requests` / `approval_actions` 对 authenticated 的直接 INSERT。
- 重写 `create_approval`，按类型验证角色、对象访问权、workspace 和状态。
- 重写 `request_refund`、`request_marketing_contact_export`、`resolve_data_quality_issue`。
- 加固 `create_quote`、预约关联对象和导入 UPDATE/MERGE 的 workspace 校验。
- 对关键实体补充 workspace 组合唯一键/外键或约束触发器。
- 建立受控员工状态更新 RPC、登录限流 RPC、worker heartbeat 与运维指标函数。

### 验收

- 低权限用户不能直接插入审批。
- 跨 owner/team/workspace UUID 全部失败。
- 旧退款函数不存在。
- maker/checker、AAL2 和 required role 不可绕过。

## 第二阶段：API 与服务边界

### 任务

- 新增 API 专用 `requireApiUser` / `requireApiRole` / `requireApiAal2`。
- 定义统一错误体：`{ error: { code, message, requestId, details? } }`。
- 定义共享 query parser，限制 page/pageSize、枚举和日期范围。
- 建立共享客户端请求方法，统一 JSON 解析、401/403、网络错误和 Abort。
- 改造所有 API route 的身份认证和关键参数校验。
- 健康检查拆分 liveness/readiness，加入 REST、数据库、队列和 worker 新鲜度。

### 验收

- 未登录 API 返回 401 JSON；越权返回 403；MFA 不足返回 403/MFA_REQUIRED。
- malformed query 返回 400，不产生 500。
- 前端不会把登录 HTML 当 JSON。

## 第三阶段：身份生命周期与防滥用

### 任务

- 员工 membership/team 更新使用事务 RPC。
- Auth metadata 更新失败时执行补偿并记录待修复状态。
- 停用员工后显式撤销会话。
- 登录限流改为数据库原子计数；本地 Map 仅作为无数据库时的开发保护。
- 安全审计记录账号状态、角色、会话撤销与限流事件。

### 验收

- 任一步骤失败均不会留下无提示的半更新。
- 停用用户的现有会话不能继续访问 CRM。
- 多实例登录限流共享状态。

## 第四阶段：可信 UI/UX

### 任务

- 删除固定关系分数，使用真实 workspace 指标或明确空状态。
- 用户菜单和设置页使用真实 MFA、邮箱验证、账号状态。
- 设置页改为显式 loading/ready/error 状态机，失败时禁止保存并支持重试。
- 全局搜索实现 Ctrl/⌘K、loading/error、方向键、Enter、Escape、外部点击和焦点恢复。
- 通知、用户菜单和导航分组补齐弹层语义和键盘行为。
- 通用表格增加完整错误/重试、可发现筛选、合理分页和移动字段标签。
- 删除学生、家庭和未启用 AI 文案/隐藏字段。
- 合并 CSS 审计补丁，建立清晰的响应式和焦点层。

### 验收

- 不显示无法由后端证明的指标或安全状态。
- 375px 无横向溢出，移动卡片字段可辨识。
- 核心操作全键盘可达，焦点可见且弹层正确关闭。

## 第五阶段：运营与业务能力

### 5.1 权限解释器

- 按角色、MFA、workspace、team、owner 和资源状态输出 allow/deny 与原因。
- 页面只解释服务端计算结果，不在浏览器推断权限。

### 5.2 工作流运维中心

- 聚合审批、退款、生成任务、日历投递、通知 outbox、导入批次和数据质量问题。
- 显示队列数量、最老任务、失败、卡死、最近 worker 心跳和 SLA。
- 管理员可对允许的失败任务执行受控重试。

### 5.3 客户活动记录器

- 在客户 360 中记录电话、邮件、会议、拜访和备注。
- 强制摘要、发生时间和下一步行动；遵守对象级权限并写审计。

### 5.4 续约 Playbook

- 为合同建立续约阶段、风险、下一步行动、负责人和到期 SLA。
- 自动识别 90/60/30/14/7 天窗口和逾期未处理项。

### 5.5 商机阶段守卫

- 开放阶段要求下一步行动和预计成交日。
- WON 要求成交证据；LOST 要求原因。
- 阶段变更写入历史并支持预测准确率统计。

### 5.6 去重预览和导入 Dry Run

- 返回字段级 before/after、冲突、受影响关系和推荐主记录。
- 导入执行前汇总 CREATE/UPDATE/MERGE/SKIP/INVALID。
- 合并和导入继续使用幂等、批次、审计和可冲突回滚。

## 第六阶段：集成、分析与规则建议

### 集成中心

- 保存 provider、状态、同步方向、游标、最近同步和错误，不保存浏览器端密钥。
- 建立签名 Webhook 收件箱、幂等键、处理状态、重试和审计。
- 在没有供应商凭据时显示“未连接”，不提供假同步按钮。

### 产品和汇率

- 产品包与选配项使用版本化关系。
- 汇率快照保留来源、时点、本位币和锁定值；历史金额不按实时汇率回写。

### Next Best Action

- 规则引擎输出证据、规则版本、优先级、有效期和建议任务。
- 用户可采纳为任务草稿或驳回并填写原因。
- 禁止建议自动执行外部通信、阶段、折扣、审批和财务动作。
