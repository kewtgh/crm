# Lumina Education CRM v2.1.0 整改与产品闭环计划

- 计划日期：2026-07-20
- 输入：`AUDIT_2026-07-20_V2.1.0.md`

## 完成定义

每项必须同时具备数据库约束/RLS、API 验证、双语 UI、错误/空/加载状态和自动化证据。
“表存在”“函数源码含关键字”不再视为业务完成；关键状态机必须通过真实执行测试。

- [x] PROG-06 增加真实“预览 → 复核 → 应用 → 重试”与未知年级 HOLD 的 pgTAP。

## Phase 5A：按角色认证与可信设备（补充需求）

- [x] AUTH-08 增加真实 Chromium “Turnstile → 账户名/密码 → 邮箱 OTP → 首次改密 → 可信设备复用”门禁。
